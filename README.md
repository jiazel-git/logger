# jzlog

一个高性能的 C++17 日志库，专注于零拷贝、异步写入和自动归档管理。

## 特性

- **零拷贝设计** - 双缓冲区技术，写入路径无锁
- **异步写入** - 后台线程处理磁盘 I/O，不阻塞主线程
- **自动归档** - 支持定时打包、压缩和过期清理
- **线程安全** - 多线程环境下安全使用
- **文件轮转** - 按大小自动切割日志文件
- **格式化支持** - 类 printf 的格式化接口
- **动态缓冲区** - 根据消息长度自动调整缓冲区大小，避免溢出
- **低延迟写入** - 条件变量通知机制，日志立即写入而非依赖超时

## 架构设计

用户代码 → CLogger → CLoggerImpl → Sink → 文件/控制台

## 代码结构

核心头文件、实现文件、测试和示例。

## 快速开始

创建构建目录后使用 cmake 配置并编译，然后运行测试和示例。

## 基本使用

包含头文件后创建日志器并记录日志，支持格式化输出。

## 启用自动归档

配置归档参数后创建支持自动归档功能的 FileSink。

## 性能特点

- **写入无锁** - 用户线程只访问写缓冲区，需加锁
- **O(1) 交换** - flush 时只交换指针，不复制数据
- **零阻塞** - 写入不被磁盘 I/O 阻塞
- **低延迟** - 条件变量通知机制，后台线程立即唤醒

### 性能指标

- **单条日志延迟** < 100ns (不含 I/O)
- **内存占用** 最小化，动态缓冲区按需分配
- **支持高并发** - 多线程安全写入

## 日志级别

TRACE = 0   // 最详细的跟踪信息
DEBUG = 1   // 调试信息
INFO  = 2   // 一般信息
WARN  = 3   // 警告信息
ERROR = 4   // 错误信息
FATAL = 5   // 致命错误
OFF   = 6   // 关闭日志
ALL   = 7   // 所有级别

## 归档功能

### 自动归档

- **每日打包** - 凌晨 2 点自动打包前一天日志
- **二级压缩** - tar 文件超过 100MB 自动使用 zstd 压缩
- **过期清理** - 自动删除超过 30 天的归档文件

### 目录结构

log/
├── current/              # 当前活跃日志
├── archived/             # 按日期归档
├── tar/                  # tar 打包文件
└── compressed/           # zstd 压缩文件

## 扩展开发

### 自定义 Sink

继承 jzlog::sinks::ISink 接口并实现虚函数，然后通过 logger.add_sink() 添加。

## 依赖

- **C++17** - 编译器要求
- **pthread** - 线程支持
- **zstd** (可选) - 日志压缩
- **std::filesystem** - C++17 文件系统支持

## 平台支持

- Linux
- macOS
- Windows (部分功能)
